thrust/data/status

Defined in src\data\event.types.js:24

The thrust/data/status event is fired for every item that returns from the queue.

NOTE: It is entirely possible for thrust/data/status** to be called afterthrust/data/stop** if several calls take to long to complete. This is intended, and potentially useful information.

Event Payload:

  • queryId String

    The internal id of the outbound queue.

  • type String

    The type of the outbound event queue (Get/Post/etc)

  • count Number

    The current completed call count for this queue.

thrust/data/stop

Defined in src\data\event.types.js:36

The thrust/data/stop event is fired when all items in the queue return, or the finishedTimeout setting elapses.

Event Payload:

  • queryId String

    The internal id of the outbound queue.

  • type String

    The type of the outbound event queue (Get/Post/etc)

  • count Number

    The (current) completed call count for this queue.
